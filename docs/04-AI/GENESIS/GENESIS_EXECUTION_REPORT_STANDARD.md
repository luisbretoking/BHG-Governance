# Genesis Execution Report Standard

## 1. Purpose

This standard defines the mandatory structure of every report generated by Genesis.

All execution reports SHALL be deterministic, reproducible, evidence-based and machine-readable.

---

# 2. Scope

This standard applies to every Genesis engine capable of producing execution outputs.

Including but not limited to:

- Repository Scanner
- Repository Auditor
- Health Model
- Planning Engine
- Certification Engine
- Future Genesis Engines

---

# 3. Mandatory Report Metadata

Every report SHALL contain:

- Report ID
- Execution ID
- Repository Name
- Repository Version
- Repository Commit
- Execution Timestamp
- Genesis Version
- Execution Engine
- Execution Mode
- Confidence Level

---

# 4. Mandatory Sections

Every report SHALL include the following sections in the exact order.

## Executive Summary

High-level description of execution.

---

## Repository Context

Repository metadata.

Repository scope.

Repository version.

---

## Execution Context

Execution intent.

Execution mode.

Executed engines.

Execution duration.

---

## Findings

Complete finding inventory.

Grouped by category.

Grouped by severity.

---

## Repository Health

Overall Health Score.

Dimension Scores.

Confidence.

Evidence.

---

## Certification Impact

Certification status.

Blocking findings.

Critical findings.

Open findings.

---

## Recommendations

Ordered remediation plan.

Priority.

Dependencies.

Estimated impact.

---

## Appendix

Evidence references.

Document references.

Repository statistics.

Execution metadata.

---

# 5. Report Quality

Reports SHALL:

- Be deterministic.
- Be reproducible.
- Be traceable.
- Be evidence-based.
- Avoid assumptions.

---

# 6. Forbidden Content

Genesis SHALL NOT include:

Speculation.

Personal opinions.

Estimated values.

Unsupported conclusions.

Non-verifiable statements.

---

# 7. Confidence Declaration

Every report SHALL declare:

Discovery Confidence

Audit Confidence

Health Confidence

Overall Confidence

---

# 8. Machine Readability

Reports SHALL preserve a consistent structure suitable for automated parsing.

Section names SHALL remain unchanged.

---

# 9. Compliance

Any Genesis report not following this standard SHALL be considered invalid for repository certification.
